# Proposal: Add Multi-Site Foundation

## Why

GEOflow 目前是单站模型。站点设置、分类、文章、任务和素材库默认共用一套全局数据，这不适合后续用一个后台管理 10-20 个精品站。

如果继续在单站模型上直接堆新站：

- 站点配置会互相覆盖
- 文章和分类会在不同域名之间串站
- slug 和 SEO 设置只能全局唯一，无法按站隔离
- 后续再改多站会更痛

## What Changes

第一阶段只落多站基础，不一次性重写后台：

1. 新增站点模型和域名映射模型
2. 给核心内容表补 `site_id`
3. 把站点设置读取改成按当前域名解析站点
4. 把前台核心内容读取改成按当前站点过滤
5. 保持当前单站数据自动归到默认站点，不打断现有运行

## Non-Goals

这次不做：

- 完整的多站后台管理 UI
- 全后台列表页和任务页的站点筛选
- 多站发布流程和多站素材运营体验
- 复杂的租户权限模型

这些留到下一阶段。
