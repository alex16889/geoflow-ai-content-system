# GEOFlow Roadmap

This roadmap keeps the public project honest: GEOFlow is a content operations system for SEO/GEO workflows, not a guarantee of ranking, indexing, or AI citations.

## Done

- Multi-site admin model with site-scoped domains, settings, categories, materials, articles, tasks, queues, and spend logs.
- DataForSEO keyword suggestion import with site-level daily budget guardrails.
- Publishing quality gates for minimum score, minimum length, and reviewable issue lists.
- Dynamic `sitemap.xml`, `robots.txt`, `llms.txt`, and `llms-full.txt`.
- Search submission queues for IndexNow, Bing URL Submission, and Baidu active push.
- SEO/GEO workbench with local readiness checks, discovery URLs, search/AI visibility records, competitor briefs, internal-link opportunities, redirects, 404 logs, and image SEO coverage.
- Structured data helpers for WebSite, Organization, Article, BreadcrumbList, CollectionPage/ItemList, and FAQPage.
- Public release audit for secrets, runtime data, generated files, and fixed default passwords.

## Next

- CSV import/export for Google Search Console and Bing Webmaster performance rows.
- Optional Google Search Console OAuth after the project has stable public domains and verified properties.
- DataForSEO-backed competitor SERP import into competitor briefs.
- Site template presets for launching a new boutique site with categories, keywords, prompts, and initial internal-link rules.
- Sitemap index and sitemap chunking when a site approaches large URL counts.
- Content similarity checks to reduce duplicate or near-duplicate article publishing.
- More complete image SEO editor in the image library detail page.

## Non-Goals

- Mass mirror-site generation.
- Automated scraping of AI answer products without explicit provider permission.
- Guaranteed Google rankings or guaranteed AI citations.
- Storing real API keys, passwords, cookies, or private deployment data in the public repository.
